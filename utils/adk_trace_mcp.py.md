# ADK Trace MCP Server (`adk_trace_mcp.py`)

## Overview
This script implements a FastMCP server named "ADK Trace Reader" designed to extract, analyze, and format execution data generated by Agentic Swarms running under the ADK (Agent Development Kit). 

## Key Tools Provided

### `get_latest_adk_session`
- **Purpose**: Reads execution traces from an SQLite session database (or a JSON fallback in `~/Downloads`).
- **Functionality**: Reconstructs the internal event loop of the agents, filtering for the most recent session implicitly if an ID is not provided. It extracts interactions, tool calls, and LLM responses, formatting them into a readable Markdown execution timeline. It also cleanly truncates large payloads (like base64 streams or massive logs) to prevent context overflows.

### `generate_session_animation`
- **Purpose**: Orchestrates the generation of interactive HTML visual representations for tracing an agent swarm session.
- **Functionality**: Triggers the `generate_animation.py` utility script located in the project root via a native subprocess. This renders the internal thought patterns, tool iterations, and state handoffs dynamically into the browser.

### `generate_global_scorecard`
- **Purpose**: Facilitates global evaluation metrics generation.
- **Functionality**: Spawns a subprocess to execute `utils/generate_global_eval_report.py`, automatically retrieving and returning the output data of the generated `GLOBAL_EVAL_SCORECARD.md` artifact.

## Security & Usage
This script primarily acts as a read-only observability layer over the `.adk/session.db` and is not designed to mutate core structures. It is extensively used to debug complex multi-agent execution loops and view the raw `args` and `responses` sent between the Swarm and Tool Server.
